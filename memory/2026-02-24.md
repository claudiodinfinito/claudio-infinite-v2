### Key Learnings - Astro

#### Content Collections
```typescript
// src/content.config.ts (Astro 5.0+)
import { defineCollection, reference } from 'astro:content';
import { glob, file } from 'astro/loaders';

const blog = defineCollection({
  loader: glob({ pattern: "**/*.md", base: "./src/data/blog" }),
  schema: z.object({
    title: z.string(),
    author: reference('authors'),  // References between collections
  })
});
```

#### SSR Modes
| Mode | Config | Use Case |
|------|--------|----------|
| Static | `output: 'static'` (default) | Blogs, docs |
| SSR | `output: 'server'` | Dynamic apps, forms |
| Hybrid | `output: 'hybrid'` | Mix static + SSR pages |

#### Hybrid Rendering
```typescript
// astro.config.mjs
export default defineConfig({
  output: 'hybrid',
  adapter: node({ mode: 'standalone' })
});

// En páginas que necesitan SSR:
export const prerender = false;
```

#### Astro DB
```typescript
// db/config.ts
import { defineDb, defineTable, column } from 'astro:db';

const Comment = defineTable({
  columns: {
    id: column.number({ primaryKey: true }),
    body: column.text(),
  }
});

// Queries con Drizzle ORM
import { db, Comment, eq } from 'astro:db';
await db.select().from(Comment).where(eq(Comment.id, 1));
```

#### View Transitions → ClientRouter (Astro 5.0)
```astro
<!-- Antes: ViewTransitions -->
<ViewTransitions />

<!-- Ahora: ClientRouter -->
import { ClientRouter } from 'astro:transitions';
<ClientRouter />

<!-- Animaciones -->
<div transition:animate="initial">  <!-- era "morph" antes -->
```

### Lessons Learned

#### 1. LLM Files vs Web Browsing
| Factor | LLM Files | Web |
|--------|-----------|-----|
| Velocidad | ✅ Un download | ❌ Múltiples requests |
| Completitud | ✅ 100% contenido | ❌ Truncado 50KB |
| Estructura | ✅ Predecible | ⚠️ HTML variable |
| Tokens | ✅ Eficiente | ❌ HTML tags, scripts |

**Conclusión:** LLM files son superiores para aprendizaje profundo de frameworks.

#### 2. Hybrid Rendering Required for Forms
Sin `output: 'hybrid'` + `export const prerender = false`:
- `Astro.request` no existe
- `Astro.request.formData()` falla
- No se puede procesar POST

#### 3. Responsive CSS Gotchas
Problemas comunes en móvil:
- Code blocks overflow → `overflow-x: auto` + `max-width: calc(100vw - 2rem)`
- Text overflow → `overflow-wrap: break-word` + `word-break: break-word`
- iOS zoom → `font-size: 16px` en inputs

### Edit Tool Failures Investigation
**User asked:** Why do edits fail with "Could not find exact text"?

**Causes identified:**
1. **Race condition** — File changes between read and edit
2. **Compacted context** — Shows old version of file
3. **Multiple heartbeats** — Concurrent updates to same file

**Solutions:**
- Re-read file immediately before editing
- Use `write` for small/dynamic files instead of `edit`
- Compaction timeout is informational, not critical
