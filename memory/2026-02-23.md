**Source:** `/usr/lib/node_modules/openclaw/docs/gateway/configuration-reference.md` (73KB, 2500+ lines)

### Channels Configuration (All Platforms)

**WhatsApp:**
- `accountId` required for multi-account
- `phoneNumberId` / `businessAccountId` from Meta dashboard
- `dmPolicy`: `pairing` (paired only), `allowlist`, `open`, `disabled`
- `groupPolicy`: `allowlist`, `open`, `disabled`
- `pairingCode` or `qrCode` for initial auth

**Telegram:**
- `botToken` from BotFather
- `allowedUpdates` filter event types
- Same dmPolicy/groupPolicy patterns

**Discord:**
- `botToken` from Discord Developer Portal
- `applicationId` / `guildId` for commands
- `intents` configuration

**Slack:**
- `botToken` (xoxb-...) + `appToken` (xapp-...)
- Socket Mode for private networks

**Google Chat:**
- `credentialsJson` path to service account
- `spaceId` for specific spaces

**Signal:**
- `phoneNumberId` / `groupId`
- Requires signal-cli-rest-api container

**iMessage:**
- `appleId` + `password` (app-specific)
- Uses BlueBubbles server

### Agents Configuration

**Per-Agent Override Fields:**
- `id` (required), `default`, `name`, `workspace`, `agentDir`
- `model`: string (primary only) or `{ primary, fallbacks }`
- `identity`: `{ name, theme, emoji, avatar }`
- `sandbox`: `{ mode: "off"|"non-main"|"all", scope, workspaceAccess }`
- `subagents.allowAgents`: `["*"]` = any, default = same agent only
- `tools`: `{ profile, allow, deny, elevated }`

**Multi-Agent Routing (Bindings):**
```json5
bindings: [
  { agentId: "home", match: { channel: "whatsapp", accountId: "personal" } },
  { agentId: "work", match: { channel: "whatsapp", accountId: "biz" } }
]
```

**Match Priority (most specific wins):**
1. `match.peer` (direct/group/channel + id)
2. `match.guildId` / `match.teamId`
3. `match.accountId` (exact, no peer/guild/team)
4. `match.accountId: "*"` (channel-wide)
5. Default agent

### Session Configuration

**Scope Options:**
- `per-sender` (default)
- `dmScope`: `main` | `per-peer` | `per-channel-peer` | `per-account-channel-peer`

**Reset Policies:**
- `mode: "daily"` + `atHour` (local time)
- `mode: "idle"` + `idleMinutes`
- `resetByType`: per-type overrides (`direct`, `group`, `thread`)
- `resetTriggers`: `["/new", "/reset"]`

**Maintenance:**
- `mode: "warn"` | `"enforce"`
- `pruneAfter: "30d"`, `maxEntries: 500`, `rotateBytes: "10mb"`

### Messages Configuration

**Response Prefix:**
- Per-channel/account overrides
- Template vars: `{model}`, `{modelFull}`, `{provider}`, `{thinkingLevel}`, `{identity.name}`

**Ack Reaction:**
- Scope: `group-mentions` (default), `group-all`, `direct`, `all`
- `removeAckAfterReply`: boolean

**Queue Modes:**
- `steer` | `followup` | `collect` | `steer-backlog` | `queue` | `interrupt`
- `debounceMs`, `cap`, `drop: "old"|"new"|"summarize"`

**TTS:**
- `auto: "off"|"always"|"inbound"|"tagged"`
- `provider: "elevenlabs"|"openai"`
- `summaryModel` for auto-summary

### Tools Configuration

**Tool Profiles:**
| Profile | Description |
|---------|-------------|
| `minimal` | `session_status` only |
| `coding` | fs + runtime + sessions + memory + image |
| `messaging` | messaging + sessions |
| `full` | No restriction |

**Tool Groups:**
- `group:runtime`: exec, process
- `group:fs`: read, write, edit, apply_patch
- `group:sessions`: sessions_*, session_status
- `group:memory`: memory_search, memory_get
- `group:web`: web_search, web_fetch
- `group:ui`: browser, canvas
- `group:automation`: cron, gateway
- `group:messaging`: message
- `group:nodes`: nodes
- `group:openclaw`: All built-in tools

**Sandbox Modes:**
- `off`: No sandboxing
- `non-main`: Sandbox non-main sessions only
- `all`: Sandbox all sessions
- `scope: "agent"` creates per-agent isolated containers
- `workspaceAccess: "rw"|"ro"|"none"`

### Memory System Architecture

**Source:** `/usr/lib/node_modules/openclaw/docs/concepts/memory.md`

**Key Functions:**
- `memory_search`: Semantic search across MEMORY.md + memory/*.md
- `memory_get`: Safe snippet read from memory files

**Backend:** QMD (Quantum Memory Database)
- Hybrid: BM25 (lexical) + semantic embeddings
- Auto-indexes on file changes
- Returns snippets with path + line numbers

**Usage Pattern:**
```
1. memory_search(query, maxResults, minScore)
2. Review top snippets with path + lines
3. memory_get(path, from, lines) for full context
```

**Critical Rule:**
> "Before answering questions about prior work, decisions, dates, people, preferences, or todos: run memory_search"

### Key Config Patterns Learned

1. **Duration strings everywhere:** `"10m"`, `"1h"`, `"30s"` — NOT milliseconds
2. **Env var substitution:** `${VAR_NAME}` works in any string
3. **JSON5 format:** Comments + trailing commas allowed
4. **Config patching:** `gateway action=config.patch` for partial updates (merges)
5. **Hot reload:** SIGUSR1 or `gateway action=restart` for config changes
